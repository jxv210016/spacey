# Screenshots & GIFs

This folder holds the images shown in the top‑level [`README.md`](../../README.md).
The repo does **not** ship images yet — drop them here and the README links will
start rendering.

## Expected files

The main README references these paths, so use these exact names:

| File | What to capture |
|---|---|
| `menu-bar.png` | The Spacey menu‑bar dropdown showing a few named Spaces (label + icon + color), with inline editing visible. |
| `mission-control.png` | Mission Control open with Spacey's custom names drawn over the Spaces Bar — the headline feature. Capture both the collapsed strip and an expanded hover state if you can. |
| `settings.png` | The Settings window with the sidebar (General / Appearance / Mission Control / Permissions / Updates / About). |

A short **GIF** of switching Spaces by name, or of names appearing as Mission
Control opens, makes a great hero image too — e.g. `demo.gif`. If you add one, link
it near the top of the main README.

## Capture tips

- Use **⌘⇧4** (then Space for a window) or **⌘⇧5** for a region/recording on macOS.
- Prefer a **clean desktop** and a neutral wallpaper so the labels read clearly.
- Capture on a **Retina** display; keep PNGs reasonably sized (downscale very large
  shots so the repo stays lean).
- For GIFs, keep them short (a few seconds) and optimized.

> Do not commit anything that exposes private information (window contents, file
> names, etc.). Spacey itself only reads the Spaces layout — keep your screenshots
> just as clean.
